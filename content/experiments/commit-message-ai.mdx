# Commit Message AI

Generate meaningful, standardized git commit messages by analyzing your code changes with AI.

## Features

- **Diff Analysis**: Examines actual code changes to understand intent
- **Conventional Commits**: Follows conventional commit specification
- **Scope Detection**: Automatically identifies affected modules/components
- **Breaking Changes**: Detects and flags breaking changes
- **Multiple Options**: Generates several commit message variations

## Conventional Commit Format

Generated messages follow the standard:
```
<type>(<scope>): <description>

[optional body]

[optional footer(s)]
```

## Supported Types

- **feat**: New features
- **fix**: Bug fixes
- **docs**: Documentation changes
- **style**: Code style changes
- **refactor**: Code refactoring
- **test**: Test additions/modifications
- **chore**: Build process or auxiliary tool changes

**Generated Messages**:

## Advanced Features

**Breaking Change Detection**:
```
feat(api)!: change user authentication endpoint structure

BREAKING CHANGE: The /auth endpoint now requires email instead of username
```

**Multi-scope Changes**:
```
feat(auth,ui): implement OAuth login with new button styles
```

**Issue Linking**:
```
fix(validation): resolve email regex pattern bug

Fixes #123
```

## Integration Options

- **CLI Tool**: `npx commit-ai` in any git repository

## Configuration

Customize generation with `.commitai.json`:
```json
{
  "maxLength": 72,
  "types": ["feat", "fix", "docs"],
  "scopes": ["auth", "ui", "api"],
  "includeBody": true,
  "linkIssues": true
}
```
